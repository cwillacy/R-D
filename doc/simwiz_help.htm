Introduction

SimWiz is an interactive GUI based wizard, for the generation of JobPro/SIPMAP skeletons, as part of the simultaneous source modelling and blending workflow (ref). The functionality provided by the wizard is comprised of the following main steps:

-        Converts SPS to SAF formats

-        Create a merged acquisition geometry

-        Generate SSF traces for forward modelling

-        Updating of trace idents post modelling

-        Blending of the data

-        Deblending o the data (optional)

-        Data QC.

 

The workflow is schematically illustrated in Figure 1.

 

 

Figure 1. Overview flowchart for the simultaneous source modelling workflow.

 

SimWiz is written in Python and can therefore be run on Windows, Linux or MacOS, with a suitable python interpreter installed. To invoke SimWiz, the user can launch the application from the command line (on Linux) e.g. on the command line:

 

/apps/sss/Anaconde3/5.0.0/bin/python3 awiz.py

 

Alternatively, for Windows and MacOS, the application can be launched from a Python IDE like Spyder, for example. In this case the user needs to open the awiz.py file in the editor and select to run the application from the IDE.

 

Differences to JobPro Wizards

SimWiz is not a JobPro wizard implementation. Its main purpose is to provide a simple means to generate the skeletons for the pre- and post-modelling acquisition design workflow. SimWiz does not build JobPro partitions or access the SQL database. In this regard it is completely separate from JobPro and therefore the user should ensure that a final check is performed of the skeletons before execution in JobPro.

 

Open Previous Wizard Setup

If SimWiz has previously been run, the parameters for the GUI will have been saved to the user directory, which is located in the SimWiz root directory. This parameter file is a CSV (comma separated variable) formatted file and can be reloaded for a new invocation for the wizard. To do this, either enter the full path to the configuration file within the text box on the introduction page or select the file from the file dialogue, which can be access by clicking the open folder icon (Figure 2).

 

Figure 2. Introduction page showing selected configuration file.

 

Wizard Execution

SimWiz is easy to use. The user may move freely forwards and backwards to each page in a linear fashion by selecting the Continue or Go Back buttons. The Cancel button will quit the wizard without the parameters being saved. One deviation from these options is presented on the final page where the Continue button changes to Done. At this point if Done is selected the wizard will go into build mode and the user can no longer return to previous pages. If the skeleton files already exist on disk, from a previously invocation of the wizard, then the user is warned with the following dialogue,

 

 

to cancel without overwriting, select No otherwise Yes will overwrite the previous files. Once the skeleton construction is complete the user is presented with the dialogue, which formally finishes execution of the wizard.

 

 

Pre-Modelling - Workflow Setup

The workflow setup page defies the parameters required to select the appropriate workflow type and some necessary users input for the JobPro wizards that are to be built.

 

Acquisition Type

Currently only the OBN acquisition workflow is implemented. Workflows for land and marine streamer acquisition may be added in the future if there is interest.

 

Source Type

Two workflows are available for the modelling workflow. The first Single will generate a workflow for a single source wavelet. This is the default option. If a survey design is required that uses two seismic wavelets, then the Mixed option can be used. Note that the mixed source type workflow requires two separate modelling instances, one for each source type.

If the mixed source type workflow is selected, then the use must enter a shot code value (SHTCOD ident) that defines the second source type. All other SHTCOD values are assumed to be associated with the first source type.

 

JobPro Revision Number

When the workflow skeletons are created, they are all labelled with the JobPro revision number at the beginning of each skeletons filename as is the requirement for JobPro. The revision number follows the usual 3 dot 2 format e.g. 123.45 or abc.de.

 

Data Pool

The output from the skeletons will be saved to a user specified data pool which can be selected from the pull-down menu in the interface. At present only a limited list is available, however, please contact the lead developer if more options are required.

Pre-Modelling - SPS File Conversion

SPS Files

The wizard currently supports the SPS file format (Shell Processing System) for the definition of the acquisition geometry. Three SPS files are required, which correspond to the source grid (.s), the receiver grid (.r) and the relational database file (.x). These files are generated in Boris Kuvshinovs Python design tool. For details of the design tool use and exporting the SPS files the reader is referred to the Shell report (SR.20.01313).

Coordinate Translation

A bulk coordinate translation can be performed on the source and receiver coordinates. To do this select the check box in the interface and add an X coordinate (X0) or Y coordinate (Y0) value to shift the data by (Figure 4). The units are assumed to be the same as the input data coordinates.

 

Figure 4. Import page which includes SPS file selection and coordinate translation options.

 

Pre-Modelling Geometry Build

Some advanced options are available for your modelling workflow:

Create mirror geometry?

Select the check box if you wish to create a mirror geometry for your survey (Figure 5). This option will redatum your vertical depths to add and extra depth shift which can be defined by the Mirror dept shift entry. Note that this does not prepare your velocity model, which needs to be undertaken separately in Smart or other software.

Apply reciprocity?

The user can apply reciprocity (swap source for receiver locations and vice-versa) by selecting this option. This would be the default for an OBN geometry, which typically has more sources than receivers so applying reciprocity would reduce the overall modelling effort.

Interpolate node depths?

The true receiver depths maybe extracted from an interpretation pick of the water bottom depth volume via the selection of a saf file. This saf file must contain the idents XREC, YREC, ZREC.

 

Figure 5. Geometry build page contains some advanced options for the desired acquisition geometry.

 

Modelling JobPro Wizards

The first few pages of the wizard cover the pre-modelling skeleton parameterization. It is assumed that the next step in the workflow will be the actual forward modelling to create the synthetic traces. Several JobPro wizards already exist for building the forward modelling skeletons. These could via the FWI wizard or RTMIG wizard in Jobpro. The user is referred to the JobPro wizard documentation for information on how to use those wizards.

 

Post-Modelling Wavelet + Noise

Following modelling some additional steps may be performed. These include options to convolve the data with a filter or add random noise (Figure 6).

Convolve data with a wavelet?

Select the checkbox and enter a WAVSAM formatted file in order to perform post-modelling wavelet convolution.

Add noise to output?

Random noise maybe added to the modelled traces via the use of the SIPMAP operation NOSADD. Noise may be added either before blending (pre) or after blending (post). Key parameters for NOSADD include the noise mode and parameters faclev, flow, iorlow, fhigh and iorhig. Details of these parameters and their use can be found in the online SIPMAP documentation.

 

Figure 6. Post-modelling wavelet and noise options.

 

Post-Modelling - Blending

Data blending requires that the user specifies the modelled trace length, sample interval and the desired output trace length. The latter value must be a multiple of 1000ms. If the start time of the data has been altered then this can be defined by entering Start time of wavelet, which is assumed to be zero by default (Figure 7).

 

Figure 7. Blending parameters.

 

Post-Modelling Deblending

An optional deblending skeleton, based on OBNPTY for OBN deblending, can be generated as part of the workflow. To do this select the generate deblending skl check box and enter the deblending parameters provided. The user is referred to the SIPMAP documentation for OBNPTY for a description of the parameters used.

 

Figure 8. Deblending page including parametrization for OBNPTY.

 

Output Options

The final options for the workflow can be defined on the output options page.

Output skl directory

By default, the skeletons are placed in the skeletons directory under the SimWiz installation directory. However, an alternative directory can be specified where the output skeletons will be written. It is more convenient to save the skeletons to the skl directory of your JobPro project, where they can be loaded directly into JobPro.

Generate SSF trace?

Option to generate or not SSF traces on output for the modelling geometry input. By default, this is always the case and is only included as an option for debugging purposes.

QC displays

Options are provided to output qc displays for the coordinate geometry, shot timing and sensor depths. Each option will generate a sperate skeletons containing operations to build qc images using GNUPLT. These images will be output to the qcimages directory in your JobPro data directory folder after the qc skeletons have been run. Examples images of typical qc displays can be seen in figures 10-13.

Sort Size

For large datasets, the default SQSORT maxrec and SRTALL maxsize parameters may need to be increased within the skeletons. To make this easy, the values can be entered in the two text boxes provided.
